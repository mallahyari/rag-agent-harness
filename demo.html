Agentic RAG — A User Story

How an AI agent answers a question over a private document corpus: not one retrieval, but a reasoning loop that decides when to search, what to search for, when to dig deeper, and refuses to answer without verifiable citations.

The loop under the hood

User question→ LLM decides next action→ tool call→ result fed back↺ (repeat ≤ 10×)→ validate citations→ answer

This is a ReAct loop: the model alternates reasoning and acting. Unlike naive RAG — which retrieves once and stuffs the results into the prompt — the agent runs as many retrieval rounds as the question needs, rewrites its own queries, and pulls extra context only when a chunk is too thin to answer from. Every tool runs sequentially; results from one search inform the next decision.
